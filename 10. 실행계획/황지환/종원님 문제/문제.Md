# 문제
1. InnoDB 모니터 활성화가 될 경우에는 통계 정보가 자동으로 갱신된다 (O / X)
2. Singleton(Value-Based 히스토그램)은 칼럼값 범위로 버킷을 관리하는 히스토그램이다. (O  / X)
3. actual time=0.007 . . 0.009에서 첫번째 0.007은 비용 계획에서 예상되는 평균 실행 시간이다. (O / X)
4. id 칼럼은 단위 SELECT 쿼리별로 부여되는 식별자 값이다(O/X)
5. 하나의 SELECT 문장 안에 여러 개의 테이블을 조인하면 조인되는 테이블 갯수만큼 id값이 증가한다. (O / X)
6. 서브 쿼리는 FROM 절 이외에서 사용되는 서브쿼리를 뜻한다(O/X)
7. ()은 MySQL에서는 FROM절에 사용된 서브쿼리를 뜻한다.
8. ()은 FROM 절의 서브쿼리에서도 외부 칼럼을 참조할 수 있게 한다.

```SQL
SELECT  
    e.first_name,  
    e.gender,  
    AVG(s.salary) AS avg_salary,         -- 급여의 평균을 계산 (집계 함수)  
    COUNT(s.salary) AS salary_change_cnt -- 급여 변동 횟수 (데이터 건수 확인용)  
FROM  
    employees e                          
    salaries s ON e.emp_no = s.emp_no    
WHERE  
    e.first_name LIKE 'A%'               
  AND e.hire_date >= '1990-01-01'      
  AND s.salary BETWEEN 80000 AND 120000
GROUP BY  
    e.first_name, e.gender               
ORDER BY  
    avg_salary DESC                      
LIMIT 10;                                -- 상위 10건만 추출
```

**A**-> Limit: 10 row(s)  (actual time=264..264 rows=10 loops=1)
    **B**-> Sort: avg_salary DESC, limit input to 10 row(s) per chunk  (actual time=264..264 rows=10 loops=1)
        **C**-> Table scan on temporary  (actual time=264..264 rows=190 loops=1)
            **D**-> Aggregate using temporary table  (actual time=264..264 rows=190 loops=1)
                **E**-> Nested loop inner join  (cost=56108 rows=45454) (actual time=0.175..252 rows=9322 loops=1)
                    **F**-> Filter: ((e.first_name like 'A%') and (e.hire_date >= DATE'1990-01-01'))  (cost=30795 rows=13883) (actual time=0.139..106 rows=9930 loops=1)
                        G-> Table scan on e  (cost=30795 rows=299733) (actual time=0.129..86.5 rows=300024 loops=1)
                    **H**-> Filter: (s.salary between 80000 and 120000)  (cost=0.865 rows=3.27) (actual time=0.0143..0.0145 rows=0.939 loops=9930)
                        **I**-> Index lookup on s using PRIMARY_KEY (emp_no=e.emp_no)  (cost=0.865 rows=9.58) (actual time=0.0133..0.014 rows=7.24 loops=9930)
